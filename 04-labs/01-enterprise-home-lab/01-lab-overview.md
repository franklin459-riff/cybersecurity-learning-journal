# 🧪 Lab Overview

## Objective

The Enterprise Home Lab was created to provide a safe environment for learning, practicing, and documenting real-world cybersecurity concepts.

Rather than relying solely on theory or certification material, this lab focuses on hands-on experience with enterprise technologies, security tools, and attack simulations.

---

## About This Lab

This lab simulates the IT infrastructure of **Mystic Technologies**, a fictional multinational software company used throughout this repository.

As the environment grows, new systems, users, services, and security controls will be introduced to reflect how a real enterprise evolves over time.

The goal is not simply to build virtual machines, but to understand how enterprise environments are designed, secured, monitored, and maintained.

---

## Learning Goals

Through this lab, I aim to:

- Build and manage enterprise infrastructure
- Practice offensive and defensive security techniques
- Understand networking and system administration
- Explore identity and access management
- Gain hands-on experience with security tools
- Document findings and lessons learned

---

## Lab Evolution

The lab will grow in phases.

### Phase 1 – Foundation

- VMware Workstation
- Kali Linux
- Metasploitable 2
- Network Configuration

### Phase 2 – Enterprise Infrastructure

- Windows Server
- Active Directory
- Windows Client
- DNS & DHCP

### Phase 3 – Modern Security

- Microsoft Entra ID
- Hybrid Identity
- Multi-Factor Authentication (MFA)
- Single Sign-On (SSO)

### Phase 4 – Security Operations

- Endpoint Security
- Vulnerability Assessment
- Security Monitoring
- Incident Investigation

---

## Documentation Approach

Every major change to the environment is documented with:

- Objective
- Environment
- Procedure
- Results
- Security Considerations
- Key Takeaways

This creates a complete record of the lab's evolution while reinforcing practical cybersecurity knowledge.
